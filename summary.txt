━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Portfolio.io is a full-stack web application that lets users create, manage,
and share professional portfolio websites in minutes. A user signs up, picks
a visual template, fills in their personal/project/experience details (or
uploads a resume and lets AI auto-fill the form), and gets a live, shareable
portfolio page instantly. They can manage all their portfolios from a personal
dashboard and delete any one they no longer need.

Core features:
  ✔  User authentication (signup / login / logout) via JWT + Supabase Auth
  ✔  Profile view and update (name, password)
  ✔  6 distinct portfolio templates (Neon Dark, Bold Minimal, White Elegance,
     Blueprint Technical, Neo Brutalist, Editorial Gallery)
  ✔  AI-powered resume auto-fill using Google Gemini 2.5 Flash (PDF & DOCX)
  ✔  AI text polishers for profile bios and project summaries ("Polish with AI")
  ✔  Recruiter Chatbot Assistant widget with vector RAG search capabilities
  ✔  Smart Job Matcher comparisons with detailed audit scorecards
  ✔  Job Description Form Auto-Tailoring & Optimizing
  ✔  One-Click PDF Resume print export layout configurations
  ✔  Create, read, update, and delete portfolios stored in Supabase (PostgreSQL)
  ✔  Personal dashboard (My Portfolios) showing all user portfolios
  ✔  Template preview with sample data before committing
  ✔  Protected routes — all dashboard/portfolio pages require login

  ─── AUTH  (/api/auth) ───────────────────────────────────────────────────────

  POST /api/auth/signup
    Purpose  : Register a new user account
    Auth     : None
    Body     : { name, email, password, confirmPassword }
    Response : { message, token, user: { id, email, name, created_at } }
    Errors   : 400 (missing fields / password mismatch / too short)
               409 (email already exists)

  POST /api/auth/login
    Purpose  : Authenticate existing user, returns JWT
    Auth     : None
    Body     : { email, password }
    Response : { message, token, user: { id, email, name, created_at } }
    Errors   : 400 (missing fields), 401 (wrong credentials)

  POST /api/auth/logout
    Purpose  : Stateless logout (client clears the token itself)
    Auth     : None
    Response : { message: "Logout successful" }

  GET  /api/auth/profile
    Purpose  : Get the logged-in user's profile info
    Auth     : Required (Bearer JWT)
    Response : { user: { id, email, name, created_at } }
    Errors   : 401 (no/bad token), 404 (user not found)

  PUT  /api/auth/profile
    Purpose  : Update name and/or password for logged-in user
    Auth     : Required (Bearer JWT)
    Body     : { name?, password?, portfolio? }   (all optional)
    Response : { message, user: { id, email, name, created_at, portfolio } }
    Errors   : 400 (no fields / password too short), 401, 404

  GET  /api/auth/portfolio/:userId
    Purpose  : Public endpoint to get user profile by their user ID
               (originally intended for public portfolio sharing)
    Auth     : None (public)
    Response : { user: { id, email, name, created_at, portfolio } }
    Errors   : 404 (user not found)

  ─── PORTFOLIO  (/api/portfolio) ─────────────────────────────────────────────

  POST /api/portfolio/create
    Purpose  : Save a new portfolio to the database
    Auth     : Required (Bearer JWT)
    Body     : {
                 personalInfo: { full_name, email_id, age, address,
                                 main_title, college_name, course_name,
                                 specialization_course_name, about_paragraph,
                                 github_username, leetcode_username,
                                 template_id },
                 techStacks:      [ { name, category }, ... ],
                 projects:        [ { project_name, project_desc,
                                      project_tech_stack[], project_github_link },
                                    ... ]   (minimum 2 required),
                 experiences:     [ { role, company_name, date_of_joining,
                                      work_description }, ... ],
                 certifications:  [ { certification_name, issuing_organization,
                                      credential_url }, ... ],
                 templateId:      "template1" | "template2" | ... | "template6"
               }
    Response : { success: true, message, portfolioId, templateId }
    Errors   : 400 (fewer than 2 projects), 401 (not authenticated)
    DB       : Inserts into tables: portfolios, tech_stacks, projects,
               experiences, certifications  (all linked by portfolio_id)

  GET  /api/portfolio/my-portfolios
    Purpose  : Fetch all portfolios created by the logged-in user
    Auth     : Required (Bearer JWT)
    Response : { success: true, count: N, data: [ ...portfolios ] }
    Errors   : 401 (not authenticated)
    DB       : SELECT from portfolios WHERE user_id = <current user>

  GET  /api/portfolio/:id
    Purpose  : Fetch full data of a single portfolio by its ID
               (personal info + all related tech, projects, experience, certs)
    Auth     : Required (Bearer JWT)
    Params   : id — portfolio UUID
    Response : { success: true, data: { personalInfo, techStacks, projects,
                                        experiences, certifications, templateId } }
    Errors   : 400 (no ID), 401, 404 (not found)

  PUT  /api/portfolio/:id
    Purpose  : Update details of an existing portfolio and re-index vectors to Qdrant
    Auth     : Required (Bearer JWT)
    Params   : id — portfolio UUID
    Body     : Same schema as /api/portfolio/create
    Response : { success: true, message: "Portfolio successfully updated" }
    Errors   : 400, 401, 403, 404, 500

  DELETE /api/portfolio/:id
    Purpose  : Delete a portfolio and ALL its related records (including Qdrant vectors) permanently
    Auth     : Required (Bearer JWT)
    Params   : id — portfolio UUID
    Security : Ownership check — only the owner can delete their portfolio
               (returns 403 if another user tries)
    Response : { success: true, message: "Portfolio successfully delete ho gaya." }
    Errors   : 400 (no ID), 401, 403 (not owner), 404 (not found), 500
    DB       : Deletes from tech_stacks, projects, experiences, certifications, then deletes from portfolios

  ─── RESUME  (/api/resume) ───────────────────────────────────────────────────

  POST /api/resume/autofill
    Purpose  : Upload a PDF or DOCX resume → AI (Google Gemini 2.5 Flash)

    Auth     : Optional (token forwarded but not strictly enforced on route)

                                   github_username, leetcode_username },
                   projects:     [ { project_name, project_tech_stack,

                   experience:   [ { role, company_name, date_of_joining,
                                     work_description }, ... ],
                   certifications: [ { certification_name,

  AUTHENTICATION FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. User signs up  → backend creates user in Supabase Auth (admin SDK)
                    → backend signs a custom JWT (payload: { sub, email })
                    → JWT stored in browser localStorage as "auth_token"

  2. User logs in   → backend verifies credentials via supabase.auth.signInWithPassword
                    → backend signs a fresh JWT
                    → JWT stored in localStorage

  3. Protected API  → frontend sends JWT as "Authorization: Bearer <token>"
                    → requireAuth middleware (backend) verifies JWT signature
                    → req.user is populated with { sub (userId), email }

  4. User logs out  → frontend clears localStorage token
                    → backend /logout is stateless (no token blacklist)

     /my-portfolios → GET /api/portfolio/my-portfolios

          → Click card → GET /api/portfolio/:id → live portfolio page

  4. DELETE PORTFOLIO
     /my-portfolios → hover card → trash icon → confirmation modal
          → "Yes, Delete" → DELETE /api/portfolio/:id
          → Card removed from UI instantly (optimistic update)

  5. UPDATE PROFILE
     /profile → /profile/edit → PUT /api/auth/profile
          → Can update display name and/or password

  HOW TO RUN LOCALLY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. Vector Database (Qdrant):
     A. Run fresh Qdrant container:
        docker run -d -p 6333:6333 -p 6334:6334 -v qdrant_storage:/qdrant/storage qdrant/qdrant
     B. Or start existing container:
        docker start romantic_swartz  (or boring_blackwell / container ID)

  2. Index existing portfolio data to Qdrant (first setup/reset):
     cd portfolio/backend
     node src/index_all.js

  3. Backend server:
     cd portfolio/backend
     npm install
     node src/server.js          (or: nodemon src/server.js)

  4. Frontend server:
     cd portfolio/frontend/vite-project
     npm install
     npm run dev

  All components must be running for RAG and Chat features to work fully.
